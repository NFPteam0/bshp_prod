## Ограничение ресурсов

Для запуска с ограничением ресурсов на контейнер приложения можно использовать следующую команду: 

```
docker compose -f docker-compose.yml -f docker-compose.resources.yml up -d
```

А ресурсы сконфигурировать в файле docker-compose.resouces.yml:

```
services:
  app:
    mem_limit: 4g # лимит на RAM
    memswap_limit: 7g # лимит дополнительной памяти на RAM (контейнер будет работать медленнее)
    cpus: "3.5" # лимит на использование ядер cpu 

  db:
    mem_limit: 2g
    memswap_limit: 3g
    cpus: "2.5"
```
Прим.: memswap_limit включает в себя mem_limit (т.е. контейнер может занимать до 7 Гб RAM
в примере, 4 из которых для обычной загрузки, а дополнительные 3 для пиковой; при использовании
дополнительной памяти контейнер может работать медленнее). Верхний предел по памяти для
контейнера составляет 7 Гб при текущих настройках.


## С ограничениями на ресурсы по умолчанию

Эта команда запустит контейнер с базовыми настройками docker, зависящими от текущей
системы и, как правило, без жестких ограничений на память:

```bash
docker compose up -d
```

# Patch 02.2026
## 1. Архитектура модели
Фундаментальное изменение архитектуры:
* CatBoost: **другой подход к ансамблированию** => более стабильный рост точности при обучении
* Добавлен **внутренний перебор гиперпараметров** => получение более высокой точности на ранних этапах без необходимости подбирать параметры (число деревьев, обучающий шаг) вручную
* **Новая идея в основе: текстовая векторная модель** предсказывает целевые поля, смотря на связанные текстовые данные (назначение платежа, наименование реквизитов), затем это предсказание корректируется, смотря на числовые и категориальные признаки (CatBoost)
## 2. Обработка текста

Добавлена **смысловая обработка текстовых данных**:
* Предобученная на текстовых данных всех баз модель **запоминает последовательности слов**. Отвечает на вопросы "Какое слово следующее?", "Какой текст по смыслу ближе к целевому?".  В основе FastText (может запоминать морфемы), сама технология аналогична автозаполнению в поисковых системах или на телефоне
* Мы добавили **кэширование на уровне слов**, в результате чего обработка текста не влияет на скорость работы
## 3. Формирование данных

Переработан алгоритм формирования датасета:
* Исправлено неправильное формирование числовых признаков (суммы, кол-во, цена)
* Теперь признаки вытягиваются "умнее": у большего числа документов мы подтягиваем большее число полей правильно
* Устранены некорректные сопоставления строк или признаков при формировании тренировочных данных
